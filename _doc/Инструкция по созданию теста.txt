1. Скопировать папки "test" "test_logs" на WEB-сервер и установить права NTFS:
	"только чтение" на папку "test" и "чтение и запись" на папку "test_logs"!
2. После установки MySQL установить пароль на пользователя "root" ("123").
	Adminer по умолчанию не подключается к MySQL без пароля, для этого добавлен плагин 
	"login-password-less", временный пароль ("mypswd") хранится в файле index.php.
3. Разрешить ему работу только на localhost.
4. Выполнить скрипт создания всех баз данных вместе с процедурами - "1.test_05.02.2018".
5. Выполнить скрипт создания пользователя "user" и наделение его необходимыми правами.

Примечание.
- Скрипт "24_group" создает таблицу для управления тестом с паролем "тестинфо24!".
- Скрипт "test_users" создает процедуры: и выполняет процедуру создания пользователя от имени которого
	будет осуществляться тестирование. кроме этого создаются процедуры:
	1)"user_grant" - создания пользователя от имени которого будет осуществляться тестирование
		и наделение его правами (выполняется после создания);
	2)"convert_utf8" - конвертирование таблиц базы данных "test" из "cp_1291" в "utf-8";
	3)"clone_group" - создание пустой таблицы "0_group" для списка группы;
	4)"clone qtest" - создание пустых таблиц "0_test_name" и "0_test_name_a" для вопросов и ответов теста.
- Скрипт "convert_utf8" зоздает и выполняет процедуру конвертирования таблиц
	базы данных "test" из "cp_1291" в "utf-8".
